## 1. 目的・スコープ

### 1.1 目的

大会申込の**締切後の事務処理フロー**を可視化・管理する機能を追加する。

**現状の課題:**
- 申込書作成や主催者への送付など、事務作業の進捗が分からない
- 抽選がある大会で当選/落選の管理ができない
- 参加費の建て替え金額が記録されていない
- 管理者が「次に何をすべきか」が不明確

**解決策:**
大会ごとの事務処理の進行状況を一目で把握し、次のアクションを明確にする専用画面を追加。

### 1.2 スコープ

| 対象 | 内容 |
|------|------|
| **対象機能** | 締切後かつ参加者がいる大会の事務処理フロー管理 |
| **対象ユーザー** | 管理者（全大会）、一般ユーザー（自分が申し込んだ大会のみ） |
| **対象外** | 締切前の大会申込機能（既存機能）、大会結果管理（既存機能）|

## 2. 申込フローのステップ定義

### 2.1 基本6ステップ

全大会共通で以下の6ステップを定義し、不要なステップは管理者が手動でスキップする。

| # | ステップ名 | 説明 | 主なタスク |
|---|-----------|------|-----------|
| 1 | **会内締切** | 景虎での申込締切を過ぎた状態 | 参加者確定 |
| 2 | **大会申込** | 申込書を作成して主催者に送付 | 申込書作成・送付 |
| 3 | **返信待ち** | 主催者からの受理連絡待ち | 返信確認 |
| 4 | **抽選結果待ち** | 抽選結果の発表待ち | 当選/落選/キャンセル待ちの確認 |
| 5 | **参加費支払い** | 会が建て替えて支払い | 支払い金額確認・振込実施 |
| 6 | **完了** | 大会当日を迎えた状態 | - |

### 2.2 ステップの条件分岐

**抽選の有無:**
- 締切後にしか抽選の有無は判明しないため、全大会に抽選ステップ(3,4)を含める
- 抽選がない大会は管理者が「ステップ3,4をスキップ」ボタンで飛ばす

**支払方法:**
- 事前振込: ステップ5を実施
- 現地払い: ステップ5をスキップ

## 3. 機能要件

### 3.1 画面構成

#### 申込フロー一覧画面

**URL:** `/application_flow`（トップバー「申込管理」メニューから遷移）

**表示対象:**
- **管理者:** 締切後 AND 参加者がいる AND 開催日前 の全大会
- **一般ユーザー:** 自分が申し込んだ大会のうち、上記条件に該当するもの

**表示内容:**

| 項目 | 内容 |
|------|------|
| 大会名 | 大会の名称 |
| 開催日 | 大会の開催日 |
| 現在のステップ | 「大会申込」「返信待ち」など |
| 参加者数 | 「12名」 |

#### 申込フロー詳細画面

**URL:** `/application_flow/:event_id`

**レイアウト:**
```
┌──────────────────────────────────────────┐
│ ● → ○ → ○ → ○ → ○ → ○                  │ ← 横向きフロー図
│ 会内  申込  返信  抽選  支払  完了          │
│ 締切                                      │
├──────────────────────────────────────────┤
│ ■ 大会情報                               │
│   第XX回北海道大学かるた大会              │
│   開催日: 2026/03/15                     │
│   締切日: 2026/02/28                     │
├──────────────────────────────────────────┤
│ ■ 次のタスク                             │
│   申込書を作成して主催者に送付する         │
│   [申込書データを見る] [CSV出力]          │
├──────────────────────────────────────────┤
│ ■ 参加者一覧（級別）                      │
│   A級(3名): 山田太郎、鈴木一郎、...       │
│   B級(5名): ...                          │

│                                          │
│   [申込書作成用データを表示] ← ボタン     │
└──────────────────────────────────────────┘
```

**表示内容:**

| 項目 | 内容 |
|------|------|
| 横向きフロー進捗図 | 現在地まで塗りつぶし |
| 大会基本情報 | 大会名、開催日、締切日 |
| 次のタスク | 「申込書を作成する」などのアクション |
| 参加者一覧（基本） | 級別、名前（漢字） |
| 参加者詳細（申込書用） | ボタン押下で表示：名前（漢字・ふりがな）、級、段位、今年度出場回数 |

**管理者専用機能:**
- ステップ進行ボタン（「次へ」「スキップ」）
- 各ステップの詳細データ入力（返信期限、抽選結果、参加費など）

**一般ユーザー表示:**
- 現在のフローステップ
- 次のアクション（「抽選結果発表まで待つ」など）
- 自分の抽選結果（当選/落選/キャンセル待ち）
- 参加者一覧（他の人も見える）

### 3.2 申込書作成機能

**機能:**
1. 画面上に参加者情報を一覧表示
2. CSVファイルとしてダウンロード

**CSV出力内容:**

| カラム | 内容 |
|--------|------|
| 氏名（漢字） | 山田太郎 |
| 氏名（ふりがな） | やまだたろう |
| 級 | A級 |
| 段位 | 3段 |
| 今年度出場回数 | 5回 |

**今年度の定義:**
- 4月1日〜翌年3月31日（学校年度）
- 公認大会(`official: true`)の個人戦(`team_size: 1`)のみカウント

**計算方法:**
```ruby
Event.where(
  kind: Event.kind__contest,
  official: true,
  team_size: 1
).where{
  (date >= Date.new(fiscal_year, 4, 1)) &
  (date < Date.new(fiscal_year + 1, 4, 1))
}.count
```

**CSVエンコーディング:**
- UTF-8 BOM付き（Excelで開く際の文字化け防止）

### 3.3 各ステップの詳細仕様

#### ステップ1: 会内締切

**自動判定条件:**
- `deadline < Date.today` AND `participant_count > 0`

**表示内容:**
- 参加者数
- 次のタスク: 「申込書を作成する」

#### ステップ2: 大会申込

**記録項目:**
- 送付日（任意）
- 送付方法（メール/郵送/Web）（任意）
- メモ欄（自由記述）

**アクション:**
- [申込書データを見る] → 参加者詳細を表示
- [CSV出力] → 申込書CSVをダウンロード
- [次へ] → ステップ3へ進む

#### ステップ3: 返信待ち

**記録項目:**
- 返信期限（任意、「未定」も選択可能）
- メモ欄（受理番号や返信内容を記録）

**期限アラート:**
- 返信期限が設定されている場合、期限が過ぎたらフロー一覧で警告表示

**アクション:**
- [返信を受け取った] → ステップ4へ進む
- [ステップ3,4をスキップ] → 抽選なしの場合、ステップ5へジャンプ

#### ステップ4: 抽選結果待ち

**記録項目:**
- 参加者ごとの抽選結果（当選/落選/キャンセル待ち）

**入力方法:**
- 手動で個別に設定（ドロップダウンで選択）

**一般ユーザー表示:**
- 自分の抽選結果のみ表示（「あなたの結果: 当選」）

**アクション:**
- [抽選結果を入力] → 参加者一覧で個別に結果を設定
- [次へ] → ステップ5へ進む

#### ステップ5: 参加費支払い

**記録項目:**
- 級別の参加費（システムデフォルトから自動取得）
- 支払期限（任意）
- 振込先情報（任意）
- 支払済みフラグ

**参加費の設定:**

`conf.rb` にシステム全体のデフォルト値を設定:

```ruby
CONF_DEFAULT_CONTEST_FEES = {
  "A級" => 3500,
  "B級" => 3000,
  "C級" => 2500,
  "D級" => 2000,
  "E級" => 1500
}
```

大会ごとに個別設定も可能（将来拡張）。

**表示内容:**
- 級別の参加費と人数
  - A級: 3,500円 × 3名 = 10,500円
  - B級: 3,000円 × 5名 = 15,000円
  - **合計: 25,500円**
- 支払期限、振込先

**建て替え記録:**
- 会が建て替えた金額として記録（後日個人から集金するため）

**アクション:**
- [支払済みにする] → 支払完了フラグをON、ステップ6へ進む
- [ステップ5をスキップ] → 現地払いの場合、ステップ6へジャンプ

#### ステップ6: 完了

**自動遷移条件:**
- 大会開催日を過ぎた場合、自動的に完了扱い

**表示内容:**
- 「大会が終了しました」

## 4. データベース設計

### 4.1 新規テーブル

#### event_application_flows

申込フロー全体の管理テーブル。

```ruby
create_table(:event_application_flows) do
  primary_key :id
  foreign_key :event_id, :events, null: false, unique: true

  Integer :current_step, null: false, default: 1  # 1〜6

  # ステップ2: 大会申込
  Date :application_sent_at                       # 送付日
  String :application_method                      # 'email', 'mail', 'web'
  Text :application_memo                          # メモ

  # ステップ3: 返信待ち
  Date :response_deadline                         # 返信期限
  TrueClass :response_deadline_tbd, default: false # 未定フラグ
  Text :response_memo                             # 受理番号等

  # ステップ5: 参加費支払い
  Date :payment_deadline                          # 支払期限
  Text :payment_bank_info                         # 振込先
  TrueClass :payment_completed, default: false    # 支払済み
  Integer :total_fee                              # 合計参加費（円）

  DateTime :created_at
  DateTime :updated_at
end
```

#### event_user_application_statuses

参加者ごとの状態管理テーブル。

```ruby
create_table(:event_user_application_statuses) do
  primary_key :id
  foreign_key :event_user_choice_id, :event_user_choices, null: false, unique: true

  # ステップ4: 抽選結果
  String :lottery_status                          # 'pending', 'won', 'lost', 'waiting'

  # ステップ5: 参加費
  Integer :fee                                    # 個人の参加費（円）
  TrueClass :fee_paid_to_organizer, default: false # 会→主催者への支払済み
  TrueClass :fee_collected_from_user, default: false # 個人→会への集金済み

  DateTime :created_at
  DateTime :updated_at
end
```

## 8. UI/UXデザイン

### 8.1 横向きフロー図

```
● ━━━ ○ ━━━ ○ ━━━ ○ ━━━ ○ ━━━ ○
会内    申込    返信    抽選    支払    完了
```

## 9. 既知の制限事項と今後の課題

### 9.1 制限事項

1. **抽選の自動判定不可**
   - 締切後にしか抽選の有無が分からないため、手動でステップスキップが必要

2. **参加費の個別設定不可（初期版）**
   - 学生割引などには対応せず、級別の一律金額のみ

3. **Excel出力未対応**
   - CSV出力のみ（将来的にExcelフォーマット対応も検討）

### 9.2 今後の拡張案

1. **大会ごとの参加費カスタマイズ**
   - 特別な大会のみ級別金額を個別設定可能に

2. **抽選結果のCSV一括インポート**
   - 主催者から受け取ったCSVをアップロードして一括設定

3. **リマインダー通知**
   - 返信期限や支払期限が近づいたらLINE/Web Push通知

4. **統計機能**
   - 年度ごとの大会数、参加費総額、出場回数ランキングなど

1. プロジェクト概要

本プロジェクトは、モダンなクロスプラットフォーム（iOS/Android）アプリケーションの開発を目的としています。直感的なフロー管理と高度なステータス追跡機能を核とし、開発効率を最大化する共通コンポーネントライブラリを活用します。

  • check_circle ユーザーの行動をタイムライン形式で可視化するフロー管理システム
  • check_circle ダークモードとライトモードの完全対応と色覚への配慮
  • check_circle ステートレスなロジック設計による高い拡張性

4. 状態ロジック

状態 (State) UI表現 動作ロジック
完了 (Completed) Primary 背景色 + Check アイコン 読み取り専用、再編集不可。タイムラインは実線で結合。
実行中 (Active) Primary ボーダー + Pulse アニメーション 現在フォーカスされているステップ。ユーザー入力が可能。
待機中 (Pending) Slate 背景色 + Lock アイコン 前のステップが完了するまで非活性。タップしても無効。

5. 実装ノート

TimelineComponent.tsx TypeScript 
interface TimelineProps {
  steps: Step[];
  currentStepIndex: number;
}

const Timeline = ({ steps, currentStepIndex }: TimelineProps) => {
  return steps.map((step, index) => (
    <StepItem 
      status={getStatus(index, currentStepIndex)}
      data={step}
      isLast={index === steps.length - 1}
    />
  ));
}

アクセシビリティ (a11y)

すべてのインタラクティブ要素には適切な aria-label を付与し、スクリーンリーダーで現在のステータス（例：「ステップ2 実行中」）が正しく読み上げられるようにすること。

パフォーマンス

タイムラインのアニメーションは Native Driver を使用し、メインスレッドへの負荷を最小限に抑える。リストレンダリングには FlashList の使用を推奨。
